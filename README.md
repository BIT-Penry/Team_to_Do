# 🎯 项目名称：Team\_to\_Do

> **项目简介**
> 本项目是北京理工大学自动化学院《软件工程导论》课程大作业，旨在实现一个支持**团队协作的DDL任务管理器**。项目基于 `HBuilderX` 和 `uni-app` 构建，支持前后端一体化开发与本地云函数模拟运行，适用于希望快速上手 Web 开发或 uni-app 的初学者。

虽然仍有诸多可优化空间，但本项目提供了从前端页面设计到后端数据逻辑的完整架构，具有较强的参考价值。

---

# 📚 项目详细说明

## 2.1 项目运行说明

* 安装并配置 HBuilderX 和必要插件（如 `uniCloud 本地调试插件`、`scss 编译插件` 等）
* 克隆或下载项目后，使用 HBuilderX 打开项目根目录
* 启用内置浏览器或连接真机/模拟器（推荐 mumu 模拟器）运行
* 使用项目中的模拟云函数完成用户注册、任务添加等业务逻辑测试

## 2.2 项目目录结构（部分展示）

由于 GitHub 无法展示完整结构，以下为核心目录结构简化展示：

```
Team_to_Do/
├── App.vue                  # 主应用组件
├── main.js                  # 入口 JS
├── manifest.json            # 应用配置
├── pages.json               # 路由配置
├── uni.scss                 # 全局样式
├── pages/                   # 页面组件
│   ├── login/               # 登录页
│   ├── register/            # 注册页
│   ├── Solowork/            # 个人任务页
│   ├── Teamwork/            # 团队任务页
│   └── ...                  # 其他页面
├── static/                  # 静态资源（图标、图片等）
├── uniCloud-aliyun/         # 云函数目录
│   ├── cloudfunctions/
│   │   ├── registerUser/    # 用户注册逻辑
│   │   ├── loginUser/       # 用户登录逻辑
│   │   ├── addtaskname/     # 添加任务
│   │   └── ...
│   └── database/            # 数据库脚本
└── uni_modules/             # 插件模块
```

## 2.3 项目使用插件

本项目整合了以下常用插件和功能模块：

* ✅ Git Blame / Git 插件
* ✅ uni-id-pages（用户身份管理）
* ✅ uniCloud 本地调试插件
* ✅ uni\_modules 插件生态
* ✅ 内置浏览器 + 终端 + uts 编译器
* ✅ SCSS / SASS 编译器支持
